# register On Sprite Direction Update Handler

* Run on sprite dirction updated, present view point to Sprite facing dirction, or which angle you see of the sprite.

```sig
Render.registerOnSpriteDirectionUpdateHandler()
```

## ~hint
* Not required, if you have used the set animations block provided.
* Used for working with other animation extensions, to set proper Image for sprite according to current seeing direction.
## ~

## Parameters
  * **dir** It is a float number, 0 ~ 1 corresponds to 0 ~ 360Â°, suggest use Math.round(dir*dirAniTotalCount)%dirAniTotalCount to get index of direction
## Returns

* none
